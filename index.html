Fuckarounditis(毫无目的地做事)

Fuckarounditis，Fuck around + itis。Fuck around，“瞎混“，”毫无目的地做事“。itis，医学后缀，一般是”炎症“或”疾病“的意思。

最近读了一篇文章，《Fuckarounditis》，作者指出了很多人在健身时，其实就在瞎混。不追踪自己的成果，没有方法，没有计划，而且没有聚焦关键动作，避开关键的深蹲和硬拉，而是在做很多无关紧要的辅助动作。虽然花了很多时间，但似乎效果不好。作者把这种随机的，无效的健身，称为 Fuckarounditis(毫无目的地做事)。

文章提出了一个清晰的，也让我很震惊的判断标准：经过两年系统训练，应该能达到以下水平：卧推体重的1.2倍，做8个标准引体向上，深蹲体重的1.6倍，硬拉体重的2倍。如果距离这个标准很远，就说明训练方法可能存在问题。

解决方案出人意料地简单：专注于几个基础的复合动作（深蹲、硬拉、卧推、引体向上），每周训练三次，每次聚焦1-2个主要动作。关键是要系统记录每次训练的重量和次数，确保持续进步。

文章是关于健身的，但它可以是关于任何事情的。fuckarounditis，在非关键事项上耗费大量时间和精力，却对真正重要的指标缺乏跟踪和专注。也就是，简单地「聚焦最高优先级的事情」，并且应该有记录，有追踪，回归基本面。

AI 没有效用函数

我发现现在想要赚钱，都是为人类创造价值，但我在想，为什么要服务人类？能不能直接服务 AI，满足 AI 的需求呢？

这就引出一个问题：如果想要通过满足AI的需求来赚钱，就必须要求AI可以成为「经济主体」。我们是否应该，真的把 AI 视为独立的经济主体，而不仅仅是工具？

而 AI 可以成为经济主体的必要条件：

  1. 有原发性欲望（内生效用函数）
  2. 能够进行资源交换
  3. 对资源有主动支配权

这里，或许就发现AI与人类的一个根本差异：AI没有欲望。

人类经济学里面，有一个很基本的概念：效用（utility），理性人是追求效用最大化的。但是，AI 没有原发性欲望，就说明，他们实际上没有真正的效用函数，他们的效用函数是人类规定的，本质上还是人类欲望的投射。也就是说，这里有一个根本的悖论：AI可能看似是在优化某个效用函数，但这个效用函数，完全是AI系统外部施加的，而不是系统内部涌现的。它们没有真正的「想要」，只有被编程的「应该」。

再思考，为什么 AI 能做到这么理性？规定了效用函数，就真的能朝着效用函数最大化的方向前进？有没有可能正是因为AI没有欲望？我们说AI没有原发性欲望，觉得这是一种缺陷。但反过来思考：人类被原发性欲望驱动，是否也是一种局限？

也许人类被欲望驱动的状态，反而是一种不够理性和自由的状态。就像我们觉得AI没有真正的欲望是局限，但实际上这可能是一种更纯粹的存在方式。而我最初提问题的出发点是赚钱，反而更可能暴露了人类的局限性。

注：上述分析，假定 AI 仍然按照人类的「经济学」进行资源配置，但实际上，AI 可能演化出自己的资源配置模式。

人类智能的新涌现：大模型时代的认知革命

人类可能正在经历一场「认知革命」，这种「认知革命」的深度可能超过「印刷术」对人类认知的影响。用AI的涌现类比，我称为「人类智能的新涌现」。

在前大模型时代，非主流的观点是稀缺的，人们很难见到奇怪的但值得深思的，挑战常规思维的的观点和想法。Peter Thiel 甚至用”What important truth do very few people agree with you on?”来筛选创业者。也因此暴论特别能吸引眼球。但现在，有了以Claude为代表的LLM，非主流的观点不再稀缺。随随便便就可以获得一大堆异端学说，例如 爱情的异端学说，友谊的异端学说，英语学习的异端学说。

在能轻而易举地接触了大量的高价值，非主流，有洞察力，尖锐的观点和想法想法时，认知进化的速度就会有质的改变。自己就会惊讶地发现，自己可以对每一个观念都发起冲击，然后发现自己居然隐藏了这么多错误的观念，这么多错误的认知。

在这样大的信息量的喂养下，就能深度激发自己的思考，获得极大的创新力。

更重要的是，「认知获取模式」，从原来的「标准化内容」到现在的「个性化激发」。以前摄入的内容，都是外界单向输入到自身，自身有什么观点和想法，就很难再和别人进行深度交流了。但现在，人类可以跟LLM进行交互。它激发你的思考，帮你扩散思维，指出你的观点中，可能存在的偏见和盲点。在在这种交互当中，思维的深度就可以得到极大扩展。

创新模式正在发生根本转变：在从「找到别人没想到的」转向「在海量可能性中找到最优组合」。这种转变可能带来创新效率的指数级提升。以前创业是要挖掘到别人没注意到的需求，投资是要找到「正确的非共识」，但现在，需要从无数种可能性中，排列组合，找到最佳解。

我还无法预料，这样的「人类智能涌现」，会给人类社会带来怎样的冲击。

应该专注于prompt而非训练模型

最近在和朋友聊天时，不只一次被建议考虑「训练模型」。

这种建议虽然出于好意，但可能忽视了以下关键问题：

首先，训练大模型，对于知识的要求很高，大部分人都没有训练模型的知识水平。而且，我们还需要专门收集我们个人的数据库，但作为个人，真的有那么大的数据吗？恐怕并没有。而 prompt，大多数人都可以写，只需要会说话就可以了。

其次，训练大模型也需要钱，需要算力。Fine-tuning, RLHF, LoRA，等等，就算真的训练了，训练后的模型能力还可能会下降，并且训练后的模型，也难以和个人知识库数据保持同步更新。而使用 prompt，成本几乎为0。

就解决实际需求而言，使用 prompt 的效果就比微调模型要好。因为 propmt 可以精准地调整输出，可以反复优化和调整，并且可以更好地利用 LLM 的涌现能力（emergent abilities）。并且，现在大的模型，输入可以有200K tokens，也就是大约10万汉字，有个人数据，完全可以直接复制粘贴进去。

我个人觉得，对于大部分人而言，认识到 LLM 的局限性（例如回答常出现的问题）+积累个人Prompt模板库，就完全够用了。如果再为LLM专门积累自身的生活数据集，效果会更加惊艳。

官网版本的调用链路： 用户输入 → Anthropic官网前端 → Anthropic API → Claude模型 → 返回结果

第三方平台的调用链路： 用户输入 → 第三方平台前端 → 第三方平台后端 → Anthropic API → Claude模型 → 返回结果 → 第三方平台后端→第三方平台前端 → 展示

按理说，多封装一层 API 并不影响大模型的性能。但是第三方平台可能倾向于更短的回答以节省 prompt；修改了 AI 安全；上下文记忆没有正确设置和处理，等等原因，最后的结果是，性能受到影响。现在 LLM 的涌现仍很复杂，输入微小的不同，就很可能大幅影响输出。

此外第三方平台可能设置了自己的 prompt，或者是 Anthropic 优化了官网版的 Claude，也影响了结果。典型例子就是302.ai。

如果使用dify.ai, langchain等中间件平台，我尚不确定对 LLM 性能有什么影响。

因此，为了体验到最佳性能，建议不要使用第三方平台的 LLM，直接在官网使用。
